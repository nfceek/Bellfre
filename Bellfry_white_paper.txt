# Bellfry

## An Open Discovery Layer for AI Agents

**Version 0.1 – Concept White Paper**

---

# Executive Summary

Artificial intelligence is rapidly evolving from isolated chatbots into autonomous software capable of communicating with applications, businesses, services, and other AI systems. As this ecosystem grows, one fundamental question remains largely unanswered:

**How do AI agents discover one another?**

Today's web provides mature solutions for human discovery. Search engines index websites, DNS resolves domain names, and APIs allow applications to exchange information. However, there is no widely adopted, lightweight mechanism that allows AI agents to advertise their capabilities, publish connection information, and become discoverable by people, software, or other AI agents.

Bellfry is an exploration of that missing layer.

Rather than building another language model or chatbot, Bellfry focuses on **agent identity and discovery**. It proposes a simple, open approach for describing what an AI agent is, what it can do, how it can be contacted, and how it interoperates with other systems.

The long-term vision is to provide a common foundation that enables AI agents to participate in an open ecosystem while remaining independent of any specific AI vendor or technology platform.

---

# The Problem

The modern Internet was designed primarily for people.

Businesses publish websites.

Search engines index pages.

Browsers display content.

Humans read information.

The emerging AI ecosystem works differently.

Increasingly, software will not simply retrieve web pages—it will seek services capable of performing tasks.

Examples include:

* Scheduling appointments
* Ordering products
* Looking up inventory
* Checking business hours
* Retrieving technical documentation
* Calculating mortgage estimates
* Booking home services
* Coordinating logistics
* Querying specialized knowledge

While APIs expose functionality, they do not solve the problem of discovery.

An AI agent must first know that another agent exists before it can communicate with it.

Today this is generally solved by manual configuration or application-specific integrations.

That approach does not scale.

---

# Vision

Bellfry proposes that every AI-enabled service should be capable of publishing a lightweight machine-readable identity describing:

* Who the agent represents
* What capabilities it provides
* Which communication protocols it supports
* How authentication works
* Where requests should be directed
* Version and compatibility information

Rather than replacing existing APIs, Bellfry complements them by making services easier to discover and understand.

In this model, Bellfry acts as a beacon.

It announces an agent's presence without dictating how the agent itself is implemented.

# The Bellfry Manifest

At the center of the proposal is a simple concept:

An **Agent Manifest**.

The manifest is a lightweight document that describes an agent's public identity.

Example information may include:

* Agent name
* Organization
* Description
* Industry
* Capabilities
* Supported protocols
* Authentication methods
* Contact endpoint
* Version
* Metadata
* Security information

This document is intentionally lightweight and extensible.

Its purpose is discovery rather than execution.

# Discovery

Bellfry envisions a searchable registry capable of indexing agent manifests.

Rather than searching for websites, applications could search for capabilities.

Examples:

* Find an HVAC scheduling agent
* Locate a restaurant ordering service
* Discover a mortgage qualification assistant
* Search for inventory lookup services
* Find weather forecasting providers

The registry becomes a catalog of services rather than a catalog of documents.

---

# Why Discovery Matters

Future software will increasingly orchestrate multiple specialized AI systems.

Rather than one large model attempting to answer every question, systems may delegate work to purpose-built agents.

A customer planning a home purchase could interact with:

* Mortgage agent
* Insurance agent
* Home inspection agent
* Utility provider
* Moving company
* County tax information

Each system contributes specialized knowledge while remaining independently managed.

Bellfry seeks to simplify the process of discovering those services.

---

# Design Principles

Bellfry is guided by several core principles.

## Open

The specification should be publicly documented and available for anyone to implement.

---

## Lightweight

Publishing an agent should require only a small manifest rather than a complex deployment.

---

## Vendor Neutral

Bellfry should not depend on any single AI provider or language model.

Whether an agent uses GPT, Gemini, Claude, open-source models, or future technologies should be irrelevant.

---

## Extensible

The specification should evolve without breaking existing implementations.

---

## Simple

Developers should be able to publish an agent in minutes rather than days.

---

# Relationship to Existing Standards

Bellfry does not seek to replace existing communication protocols.

Instead, it complements them.

Existing technologies such as REST APIs, Model Context Protocol (MCP), Google's Agent-to-Agent (A2A) initiatives, and future interoperability standards can all coexist with Bellfry.

Bellfry focuses on discovery.

Those technologies focus on communication.

Together they provide a more complete ecosystem.

# Conclusion

Artificial intelligence is changing how software communicates.

The next generation of applications may rely less on manually configured integrations and more on dynamic discovery of specialized services.

Bellfry explores one possible foundation for that future.

By providing a lightweight, open mechanism for publishing identity and capabilities, Bellfry seeks to make AI agents easier to discover, easier to integrate, and easier to trust.

The project is intentionally small in scope but ambitious in vision.

Rather than building another AI model, Bellfry aims to help intelligent systems find one another—creating an open discovery layer that can evolve alongside the next generation of AI applications.

The future Internet may not simply connect documents.

It may connect intelligence.

Bellfry is an exploration of how that journey might begin.
